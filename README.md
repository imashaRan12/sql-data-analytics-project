# 📊 SQL Data Analysis Project

## 📌 Project Overview

A comprehensive collection of SQL scripts for data exploration, analytics, and reporting. These scripts cover various analyses such as database exploration, measures and metrics, time-based trends, cumulative analytics, segmentation, and more. This repository contains SQL queries designed to help data analysts and BI professionals quickly explore, segment, and analyze data within a relational database. Each script focuses on a specific analytical theme and demonstrates best practices for SQL queries.

### Key Objectives

- **Understand** database structure, schema, and relationships
- **Explore** dimensions, dates, and measures for complete data context
- **Quantify** data magnitude and distribution across categories
- **Rank** entities based on performance metrics
- **Track** changes and cumulative trends over time
- **Analyze** performance benchmarks and part-to-whole relationships
- **Segment** data into meaningful categories
- **Generate** consolidated reports for customers and products

## 🧭 Project Roadmap

![Project Diagram](./Docs/Project_diagram.drawio.png)

The image above outlines the step-by-step progression of this project:

| Phase  | Focus Area                 | Description                                                                                         |
| ------ | -------------------------- | --------------------------------------------------------------------------------------------------- |
| **1**  | **Database Exploration**   | Explore database structure, list tables and schemas, inspect column metadata                        |
| **2**  | **Dimensions Exploration** | Explore structure and attributes of dimension tables                                                |
| **3**  | **Date Exploration**       | Determine temporal boundaries, understand historical data range                                     |
| **4**  | **Measures Exploration**   | Calculate aggregated metrics (totals, averages), identify trends and anomalies                      |
| **5**  | **Magnitude**              | Quantify data and group results by dimensions, understand distribution across categories            |
| **6**  | **Ranking**                | Rank items based on performance metrics, identify top performers and laggards                       |
| **7**  | **Change Over Time**       | Track trends, growth, and changes in key metrics over time for time-series analysis and seasonality |
| **8**  | **Cumulative Analysis**    | Calculate running totals and moving averages, track cumulative performance over time                |
| **9**  | **Performance Analysis**   | Measure product/customer/region performance over time, benchmark high-performing entities           |
| **10** | **Part to Whole**          | Compare performance across dimensions or time periods, evaluate differences between categories      |
| **11** | **Data Segmentation**      | Group data into meaningful categories for targeted insights (customers, products, regions)          |
| **12** | **Reporting**              | Consolidate key customer metrics and behaviors, consolidate key product metrics and behaviors       |
